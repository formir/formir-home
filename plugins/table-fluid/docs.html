What is Table fluid?

Table fluid adds your table ability to display on mobile devices. Table will display as many columns that will fit on the narrow screen and the user can switch between columns using navigation.

Table fluid is RWD (Responsive Web Design or Fluid Design) extension to basic html table written in pure CSS3 and HTML. It is part of Formir Framework but can operate independently with other table styles.

Table fluid will fully work only with CSS and HTML. All of the logic is based on CSS and <input/> tag elements, that are required.

What Table Fluid give me?

Possible display on screen only those columns that will fit on your screen. You can very easily set up your Own breakpoints and number columns to display in that breakpoint.

In a device with a narrow screen (ex. tablet), your table that has 12 columns will display only 4 columns and all others will be slide using navigation buttons and pagination.

This plugin also contains a more cool feature for table, for example Mobile view for table, zebra mode, hover style on row and column, toggle row or body section of table. Look at examples.

Installation

Use LESS or SASS compiler and set up your own table fluid.

This plugin is written in LESS (and optional SASS) dynamic preprocessor style sheet language (CSS).

It is part of Formir CSS Framework. Formir gives you an idea of how to neatly sort and attach style files before compiling them in your project during the development phase.
You can use this approach or attach the only source of table fluid plugin formir-table-fluid.less. Table component form formir framework is required.

Choice LESS or SASS

In this documentation, we focus on LESS preprocessor, but the same file structure and mixin/variables are used for SASS (scss), to know how install & compile sass file go to Sass documentation

Prepare files

Start with copy all files to your root direction:

        
src                        - your application root path
  less                     - your less direction
    formir
      core
        components
          table.less       - this is part of basic table style
        helpers
          display.less
        mixins             - less mixins
          display.less
          flex.less
          functions.less
        plugins
          table-fluid.less  - all style and settings for table fluid plugin
        setup
          classes.less      - classes setup for formir framework components
          core.less         - core setup (basic variables with breakpoins etc.)
          variables.less    - variables for formir componets
    formir-table-fluid.less - your root file that imports all of other files

Include Style

Include formir-table-fluid.less into your project.

In this step, you can include formir-table-fluid.less to your main LESS file, or move the content of formir-table-fluid.less to your main (manifest/root) LESS file.

Use developer mode of less library, or compile less using backend (ex. webpack).
To use .less file on your page look at less configuration page.

Add in your <head> tag with your HTML file on developer mode:

        
<head>
  <link rel="stylesheet/less" type="text/css" href="less/formir-table-fluid.less" />
  <script src="js/less.min.js"></script>
</head>

In production mode, you can also use LESS npm package to compile your own .min.css file. Use lessc command in bash, for example:
$ npm install -g less
$ lessc -x less/formir-table-fluid.less css/formir-table-fluid.min.css

Setup LESS / SASS

Table fluid is fully configurable, each of colors, breakpoints etc. you can change in less file less/formir/plugins/table-fluid.less.

Base Table formir component is configurable with others framework components in dir: less/formir/setup/core.less and less/formir/setup/variables.less.

See all configurations of LESS file.

Setup core less variables

  
@table-fluid-mobile-max-width: 1000px; // above this browser width fuild table will off
@table-fluid-mobile-min-width: 0; // below this browser width fuild table will off, if you will use also mobile mode for fluid table set min-width to @table-mobile-max-width otherwise use 0


@table-fluid-columns:     6,       5,      4,       2,       1; // number of columns to show in breakpoint
@table-fluid-breakpoints: 900px,   700px,  600px,   500px,   @table-fluid-mobile-min-width;  // width for media query breakpoints
@table-fluid-max-columns: 10;  // maximal columns number that will be supported, use as lowers number as you need

@table-fluid-restart-include: false;   // change this to: true, if you include JS script to .checked() first input (toogle) when window resize, when not user need to click 'restrat' button to set first column to show.
@table-active-method: checked; // checked / active  ( 'active' for JS framework only.)

Width screen to turn fluid table off

Not all of windows screen width size is good to show fluid table - to turn off plugin and use default table set this two variables: @table-fluid-mobile-max-width and @table-fluid-mobile-min-width

If you will use table fluid also with mobile support form base table style component, you need to set @table-fluid-mobile-min-width equal to @table-mobile-max-width, this set will turn off table fluid when a mobile view is turned on.

Break points and columns number

This setup is most important, you can set in two variables for each step, number of columns that will show according width of screen.

In our example between two width of screen 1000px and 900px will show 6 columns, between 900px and 700px will show 5 columns, 700px - 600px: 4, 600px - 500px: 2, 500px - 0px: 1 column.

Maximum columns supported

This number of columns will be supported by plugin, use as lowers number as you need.

Need tabel fluid on React or other JS Framework?

You can use JS framework, in that case you cont neet input checkboxes, and only toggle active class in fluid-toggle divs to ster your fluid, we give your free hand.

Setup HTML

Your table fluid will look like on this example:

        
          {% include plugins/table-fluid/docs/sandbox.html %}

There are a 4 elements of table fluid plugin in your html:

  1. Set count columns class that matches your table table-column-{number}.
  2. Inputs - add as many tags <input type="radio" id="your_control_id" name="your_table" /> as there are columns in your table, each input must have a unique id for all page, all neet to have same name attribute, first neet to have checked="true" attribute
  3. Labels arrow - add as many tags <label class="table-arrow" for="your_control_id"></label> as there are columns in your table, each label must have the same for as his control input.
  4. Labels dots - add as many tags <label class="table-dot" for="your_control_id"></label> as there are columns in your table, each label must have the same for as his control input.
Warning: One JS script is not mandatory but will help your table fluid works better. After screen size will change table fluid neet to be reset and first <input type="radio" id="your_control_id" name="your_table" /> need to be set as checked. We have some walkaround for this problem. If you noot include this JS user needs to click CTA button that will show in the navigation bar to reset the fluid table.
